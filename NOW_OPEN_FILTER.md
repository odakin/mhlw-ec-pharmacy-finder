# 「今対応可能」フィルター — 設計指示書

**注**: 当初「今開いている」として設計したが、最終実装では「時間外対応あり」チェックボックスと統合し「**今対応可能**」にリネーム。ラベル検討の経緯はこの文書に残すが、最終仕様はチェックボックス名「今対応可能」、バッジ4色体系（緑「営業中」/ 青「時間外対応可」/ 灰「営業時間外」/ 琥珀「営業状況不明」）。

## なぜ今実装するのか

### 経緯

DESIGN.md §2 で「現在営業中」フィルターは **実装しない** と判断していた。理由は3軸評価:

1. **情報の出所**: 営業時間パーサーの推定（二次加工）
2. **代替の有無**: パース不能の薬局が最寄りかもしれず、代替なし
3. **除外しない場合の害**: なし。各カードに営業状況を表示する現状の設計で十分

### 前提の変化

営業時間パーサーの大幅改善（2026-03-23）により、カバー率が変わった:

| | 判断時 | 現在 |
|---|---|---|
| 薬局 | 97.1%（292件不明） | **98.2%**（183件不明） |
| 医療機関 | 0%（未対応） | **88.3%**（364件不明） |

### 設計の発見: 「不明バッジ」が3軸評価を変える

従来の分析は「パース不能施設を隠す or 全部見せるか」の二択だった。第3の選択肢が核心:

**「確実に閉まっているもの」だけ隠し、「不明」は「営業状況不明」バッジ付きで残す。**

この設計で3軸を再評価:

1. **情報の出所**: フィルターが隠すのは「パース成功 AND 営業時間外」のみ。パーサーの推定（二次加工）だが、パース成功したレコードの時間判定は正確
2. **代替の有無**: パース不能施設は隠さない。「営業状況不明」バッジで表示される。ユーザーは電話確認して判断できる
3. **除外しない場合の害**: 東京都で夜10時に検索 → 50件中40件が「営業時間外」。10件の「営業中」を探すスクロールは、モバイルで緊急時のユーザーにとって実害がある

→ **3軸すべてで条件が改善。** ユーザー起動型フィルター（デフォルトOFF）として実装する。

## 設計思想

### 原則: 「確実に閉まっている」だけを隠す

| レコードの状態 | フィルターON | フィルターOFF |
|--------------|------------|-------------|
| パース成功 + 営業中 | **表示**（緑バッジ） | 表示 |
| パース成功 + 営業時間外 | **非表示** | 表示 |
| パース不能 | **表示**（琥珀「営業状況不明」バッジ） | 表示（生テキスト） |

フィルターが新たなリスクを追加しない理由:
- 「パース成功 + 営業時間外」を隠すリスク（実は臨時営業）は、カード上の「営業時間外」表示と同じ情報に基づく。フィルターの有無でリスクは変わらない
- 「パース不能」を隠さないので、最寄り施設が消えることはない

### ラベル: 「今開いている」

他の候補と比較:

| ラベル | 却下理由 |
|--------|---------|
| `営業時間内のみ` | 「のみ」が不明バッジ付き表示と矛盾。硬い |
| `営業時間外を除く` | 「時間外対応あり」と紛らわしい |
| `閉まっている施設を除く` | 否定の否定で認知負荷高い。長い |
| `営業中のみ表示` | 「のみ」が不明バッジと矛盾 |

**「今開いている」を選んだ理由**:
- 3文字。モバイルで一瞬で読める
- 「のみ」がないので不明バッジとの矛盾がない
- Google Maps の「営業中」フィルターと同じ慣習。ユーザーが期待する動作
- 緊急時のユーザーの頭の中にある言葉（「今開いているところを探したい」）そのもの

### 「営業状況不明」バッジの設計

- **色**: 琥珀（`#b90` 系）。「営業中」緑・「営業時間外」灰と明確に区別
- **表示タイミング**: フィルターON/OFF に関係なく常時表示。パース不能であることは常に有用な情報
- **ラベル**: 「営業状況不明」。「不明」だけだと何が不明か曖昧
- **併記**: バッジの下に生テキスト（元の営業時間記載）を表示。ユーザーが自分で判断できる

### パフォーマンス: パース結果キャッシュ

`normalizeHoursText()` + `parseHours()` は正規表現の塊。13,000件にフィルター実行するたびに呼ぶと遅い。

**解決**: データロード時にパース結果をレコードにキャッシュ（`r._hoursParsed`）。パースは時間非依存なので1回計算すれば十分。フィルター時は「キャッシュ済みパース結果 + 現在時刻」で `isOpen` を判定 — 配列走査と数値比較だけなので瞬時。

### 薬局・医療機関の両方に適用

既存の4フィルター（事前連絡、時間外対応、女性薬剤師、個室）は薬局専用。「今開いている」は **薬局と医療機関の両方** に適用する。営業時間は両方にある情報であり、分ける理由がない。

## 実装仕様

### フィルター挙動

```
filterON && info.parsed && !info.isOpen → 非表示
それ以外 → 表示
```

### URL パラメータ

`?open=1` — 既存の `nocall`, `after`, `female`, `room`, `clinic` と同列。

### チェックボックスの配置

```
☐ 事前連絡「要」を除く  ☐ 時間外対応あり  ☐ 女性薬剤師がいる
☐ 個室あり  ☐ 今開いている  ☐ 常時在庫ありの医療機関も表示
```

## 除外事項

- **自動リフレッシュ**: フィルター結果は検索時点の時刻に基づく。分が変わっても自動更新しない（ユーザーが再検索すれば更新される）
- **祝日の特別扱い**: 既存の祝日判定（`getJapaneseHolidays()`）に乗る。追加実装なし
- **デフォルトON**: しない。DESIGN.md §2 のデフォルト除外の3軸評価は依然として「実装しない」が正解。ユーザー起動型に限定
