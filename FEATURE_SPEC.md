# mhlw-ec-pharmacy-finder 機能追加指示書

## 概要

緊急避妊薬の薬局検索サイト https://odakin.github.io/mhlw-ec-pharmacy-finder/ に以下の機能を追加する。
競合サイト（EPARKくすりの窓口 https://www.kusurinomadoguchi.com/s?is_ecp=1
）との差分を埋めつつ、既存の強み（厚労省公式データ10,000件超、女性薬剤師フィルター、事前連絡不要フィルター等）は維持する。

**制約**:
- GitHub Pages での静的ホスティング。サーバーサイド処理なし
- 外部APIキーが必要なサービス（Google Maps Platform等）は避け、無料・APIキー不要のものを優先
- 既存のUI/UXを壊さない。既存機能（都道府県選択、フリーワード検索、チェックボックスフィルター）はそのまま残す

---

## 機能1: 地図表示

### 要件
- 検索結果の薬局を地図上にピン表示する
- ピンをクリック/タップで薬局名・住所・電話番号等のポップアップを表示
- ポップアップ内に「Google Mapで見る」リンクを含める（`https://www.google.com/maps/search/?api=1&query={薬局名}+{住所}` 形式）。これにより写真・ストリートビュー・口コミへの導線を確保する
- 検索結果のリスト表示と地図表示を切り替え可能にする（タブまたはトグル）
- モバイルでも使いやすいレスポンシブ対応

### 技術方針
- **Leaflet.js** + **OpenStreetMap** タイル（無料、APIキー不要）
- 厚労省データに緯度経度は含まれていない。ビルド時にジオコーディングして緯度経度を付与する:
  - Nominatim（OpenStreetMap、無料、1req/sec制限）を使い、差分更新方式にする（新規・変更分のみ処理）
  - ジオコーディング結果は別ファイル（例: `geocode_cache.json`）に保持し、元データと結合して使う
  - GitHub Actionsで定期実行可能な構成にする
  - 10,128件を1req/secで処理すると約3時間。初回は時間がかかるが、以降は差分のみ
- 件数が多い場合 → **Leaflet.markercluster** でクラスタリング
- 検索結果が200件超の場合は地図タブに「検索条件を絞ると地図が見やすくなります」のガイドを表示。ただし表示自体はブロックしない（クラスタリングで対応）
- 初期表示（検索前）では地図は非表示でよい

## 機能2: 現在地からの距離・近い順ソート

### 要件
- 「現在地から近い順」ソートボタンを追加
- ボタン押下時にブラウザの Geolocation API で現在地を取得し、各薬局との距離を計算してソート
- 各薬局の行に「約○○km」の距離表示を追加
- 位置情報の取得に失敗した場合（拒否/非対応）は適切なエラーメッセージを表示し、既存のソート（表示順）にフォールバック
- 地図表示モードでは、現在地もピン（色を変える）で表示する

### 技術方針
- `navigator.geolocation.getCurrentPosition()` を使用
- 距離計算は Haversine 公式（直線距離）で十分。ルート距離は不要
- 位置情報の許可ダイアログが出ることをユーザーに事前に伝えるUI（例:「現在地を使用して近い順に並べます。位置情報の許可が必要です。」）
- ソート切替: デフォルト（元の順序）/ 近い順 をトグルで切り替え
- 距離計算には緯度経度が必要なので、機能1のジオコーディングが前提

## 機能3: 営業中かどうかのリアルタイム表示

### 要件
- 現在の曜日・時刻に基づいて「営業中」「営業時間外」のバッジを各薬局に表示
- 可能であれば「あと○時間で閉店」「○時から営業」の補足情報

### 技術方針
- 厚労省データの `hours` フィールドに営業時間が含まれている（空欄は177件/10,128件）
- これをパースして JavaScript の `Date` で現在時刻（JST固定）と比較
- **フォーマットの揺れが非常に大きい**（6,932種類）。主なバリエーション:
  - 区切り: `-`, `～`, `〜`, `~`, `‐`（全角ハイフン、波ダッシュ、チルダ等が混在）
  - コロン: `:` と `：`（全角）が混在
  - 曜日区切り: `､`, `、`, `,`
  - 曜日範囲: `月-金`, `月～金`, `月〜金`
  - 個別曜日: `月・火・木・金`
  - 代表的パターン: `月-金:9:00-18:00､土:9:00-13:00` / `月～金：9:00-18:00、土：9:00-13:00`
- パース不能の場合は「営業時間: [生データ表示]」にフォールバック。無理にパースして誤判定するより、生データ表示のほうが安全

### 注意
- 営業時間データの精度は厚労省データに依存する。祝日・臨時休業は反映できないので、「※実際の営業状況は店舗にご確認ください」の注記を必ずつける

## 機能4: Google Mapリンク（各薬局への外部導線）

### 要件
- 各薬局の検索結果行に「Google Mapで見る」アイコンまたはリンクを追加
- クリックで Google Maps が開き、該当薬局の場所・写真・口コミ・ストリートビューが確認できる

### 技術方針
- リンク形式: `https://www.google.com/maps/search/?api=1&query={encodeURIComponent(薬局名 + ' ' + 住所)}`
- APIキー不要。ブラウザで Google Maps を開くだけ
- 地図表示（機能1）のポップアップ内にも同じリンクを配置

## 実装の優先順位

1. **機能4**（Google Mapリンク）— 最も簡単。すぐ効果がある
2. **機能1**（地図表示）— ジオコーディングスクリプトの作成が必要。最大の工数
3. **機能2**（近い順ソート）— 機能1の緯度経度データに依存
4. **機能3**（営業中表示）— `hours` フィールドのパース。フォーマットの揺れに注意

## データ構造（調査済み）

- **緯度経度: なし** → ジオコーディングが必要（方針2を採用）
- **営業時間: あり** → `hours` フィールド。フォーマット例: `月-金:9:00-18:00､土:9:00-13:00`。パース可能
- **データ読み込み**: `docs/data.json` を fetch。構造は `{ "meta": {...}, "data": [{...}, ...] }`。10,128件
- **公開ディレクトリ**: `docs/`（GitHub Pages）
- **フロントエンド**: `docs/index.html` + `docs/app.js` + `docs/style.css`（バニラJS、フレームワークなし）

## その他の判断

- **ネット予約連携**: 薬局側のシステムが必要なため対象外
- **薬局の写真直接表示**: データソースがなく、APIコストも非現実的なため対象外。Google Mapリンク（機能4）で代替
- **口コミ表示**: 同上。Google Mapリンクで代替
